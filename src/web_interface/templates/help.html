🚀 Bienvenue sur RedForge v2.0

Nouvelle version

Installation

# Cloner le dépôt
git clone https://github.com/Elfried002/RedForge.git
cd RedForge

# Installer les dépendances
sudo ./install.sh

# Lancer RedForge (interface web)
sudo python3 run.py

# Accéder à l'interface
http://localhost:5000

Première utilisation

  1. Ajoutez une cible dans le tableau de bord
  2. Choisissez votre mode d'attaque :
    • Attaques simples - Pour des tests ciblés
    • Multi-attaques - Pour des tests complets
    • Opérations APT - Pour des simulations avancées
  3. Configurez le mode furtif si nécessaire
  4. Lancez l'attaque et surveillez les résultats en temps réel
💡 Nouveauté v2.0 : Support des attaques multiples, mode furtif avancé avec TOR/proxies, et opérations APT complètes!

📋 Prérequis système

  • OS : Kali Linux 2024.1+ / Parrot OS 6.0+ / Ubuntu 22.04+
  • RAM : 4GB minimum (8GB recommandé pour APT)
  • CPU : 2 cœurs minimum (4 cœurs pour multi-attaques)
  • Storage : 10GB libre (20GB pour wordlists)
  • Network : Accès internet pour mises à jour et wordlists
  • TOR : Optionnel pour mode furtif

📚 Attaques Multiples

Nouveau

Principe

Les multi-attaques permettent d'exécuter plusieurs attaques simultanément ou séquentiellement sur une même cible.

Modes d'exécution

  • Séquentiel : Les attaques s'exécutent une par une. Idéal pour éviter la détection.
  • Parallèle : Les attaques s'exécutent simultanément. Plus rapide mais plus détectable.

Exemple d'utilisation

# Sélectionnez vos attaques dans l'interface
# Configurez le mode d'exécution et le niveau furtif
# Lancez la multi-attaque

# Via API
curl -X POST http://localhost:5000/api/multi-attack \
  -H "Content-Type: application/json" \
  -d '{
    "target": "example.com",
    "attacks": [
      {"category": "injection", "type": "sql"},
      {"category": "cross_site", "type": "xss"},
      {"category": "authentication", "type": "bruteforce"}
    ],
    "execution_mode": "sequential",
    "stealth_level": "high"
  }'

🕵️ Mode Furtif

Nouveau

Niveaux de furtivité

🟢
Low
Détection facile, rapide
Délai: 0.5s
🟡
Medium
Équilibre performance/discrétion
Délai: 1.5s
🔴
High
Difficile à détecter
Délai: 3.0s
🟣
Paranoid
Maximum de discrétion
Délai: 5.0s

Techniques de furtivité

  • Jitter : Variation aléatoire des délais entre requêtes
  • User-Agents aléatoires : Rotation des user-agents
  • TOR : Routage via le réseau TOR
  • Rotation de proxies : Changement périodique de proxy
  • Mimétisme humain : Pauses et patterns humains
  • Slow Loris : Requêtes lentes et partielles
⚠️ Attention : Le mode furtif ralentit considérablement les attaques. Utilisez-le uniquement lorsque la discrétion est primordiale.

🎭 Opérations APT

Nouveau

Qu'est-ce qu'une opération APT ?

Les opérations APT (Advanced Persistent Threat) simulent des attaques avancées avec persistance, mouvement latéral et exfiltration de données.

Phases d'une opération APT

1
Reconnaissance

Collecte d'informations sur la cible

2
Initial Access

Obtention d'un premier accès

3
Persistence

Maintien de l'accès

4
Privilege Escalation

Élévation de privilèges

5
Lateral Movement

Déplacement sur le réseau

6
Data Exfiltration

Extraction des données

Opérations prédéfinies

  • Reconnaissance → Exfiltration : Opération APT complète
  • Compromission d'Application Web : Ciblage web spécifique
  • Mouvement Latéral : Propagation sur le réseau
  • Mise en place de persistance : Backdoors et scheduled tasks

Création d'opérations personnalisées

# Via l'interface, cliquez sur "Créer une opération personnalisée"
# Définissez les phases et les attaques associées
# Sauvegardez et lancez l'opération

# Via API
curl -X POST http://localhost:5000/api/apt/custom \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Mon opération",
    "phases": [
      {"name": "Recon", "attacks": ["port_scan", "service_enum"]},
      {"name": "Exploit", "attacks": ["sql_injection"]}
    ],
    "cleanup": true
  }'

❓ Questions fréquentes

🔒 Comment activer le mode furtif ?

Utilisez le toggle dans la sidebar ou l'onglet "Mode Furtif". Vous pouvez aussi utiliser Ctrl+Shift+S.

📚 Quelle est la différence entre attaque simple et multi-attaque ?

Les attaques simples ciblent une vulnérabilité spécifique. Les multi-attaques exécutent plusieurs attaques en séquence ou parallèle.

🎭 Les opérations APT sont-elles réversibles ?

Oui, le mode cleanup est activé par défaut pour nettoyer les traces après l'opération.

🕵️ Le mode furtif fonctionne-t-il avec TOR ?

Oui, activez l'option TOR dans la configuration furtive. Assurez-vous que TOR est installé et en cours d'exécution.

📊 Comment exporter les résultats d'une multi-attaque ?

Les résultats sont disponibles dans l'onglet "Résultats". Utilisez le bouton "Exporter" pour sauvegarder.

⚡ RedForge est-il compatible avec les CI/CD ?

Oui, l'API REST permet l'intégration dans vos pipelines de sécurité.

🔧 Résolution des problèmes courants

❌ TOR non trouvé

Solution : sudo apt install tor puis sudo systemctl start tor

❌ Multi-attaque bloquée

Solution : Vérifiez les logs dans /var/log/redforge/ et augmentez le timeout

❌ Mode furtif trop lent

Solution : Réduisez le niveau de furtivité ou désactivez certaines options (TOR, rotation proxies)

❌ WebSocket déconnecté

Solution : Vérifiez votre connexion réseau et redémarrez le serveur

❌ Permission denied

Solution : Lancez avec sudo pour les fonctionnalités réseau avancées
